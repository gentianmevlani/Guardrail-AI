# guardrail CLI v2.5.0 - New Features Documentation

## 🎉 Major Updates in v2.5.0

### 🎯 **Interactive Menu with Arrow Navigation**

The CLI now features a modern, intuitive interactive menu system that replaces the old numbered selection system.

#### **Key Features:**
- **Arrow Key Navigation**: Use ↑↓ arrows to navigate options
- **Visual Selection**: Current selection is highlighted with color indicators
- **Keyboard Shortcuts**: Press Enter to select, Escape to exit
- **Responsive Design**: Works in all terminal environments

#### **Usage:**
```bash
# Open the interactive menu
guardrail menu
# or
vc menu
```

#### **Menu Options:**
1. **🔍 Scan Project** - Quick access to all scanning options
2. **🔐 Authentication** - Manage API keys and authentication
3. **🚀 Reality Mode** - Browser testing with auto-installation
4. **📦 Ship Check** - Plain English audit and readiness assessment
5. **🛠️ Fix Issues** - AI-powered issue remediation
6. **⚙️ Configuration** - Project setup and templates
7. **📊 Analytics** - View scan history and statistics
8. **❌ Exit** - Exit the menu

---

### 🎭 **Reality Mode with Automatic Playwright Installation**

Reality Mode now automatically installs Playwright dependencies and browsers when needed.

#### **Enhanced Features:**
- **Auto-Detection**: Checks if Playwright is installed
- **Automatic Installation**: Installs dependencies and browsers if missing
- **User Confirmation**: Prompts before installation
- **Fallback Options**: Provides manual installation instructions if auto-install fails

#### **Usage:**
```bash
# Run reality mode with auto-installation
guardrail reality --url https://example.com --flow user-journey

# Auto-installation will trigger if Playwright is not found
```

#### **Installation Process:**
1. **Detection**: CLI checks for Playwright installation
2. **Confirmation**: User is prompted to install dependencies
3. **Installation**: `npm install playwright @playwright/test` runs automatically
4. **Browser Setup**: `npx playwright install` downloads browsers
5. **Execution**: Reality mode continues with browser testing

## 📋 **Complete Command Reference**

### **Core Commands:**

| Command | Description | New Features |
|---------|-------------|--------------|
| `guardrail menu` | Open interactive menu | ✅ Arrow navigation |
| `guardrail scan:secrets` | Scan for hardcoded secrets | ✅ Enhanced output |
| `guardrail scan:vulnerabilities` | Scan for vulnerabilities | ✅ OSV integration |
| `guardrail scan:compliance` | Run compliance assessment | ✅ Framework support |
| `guardrail sbom:generate` | Generate SBOM | ✅ Multiple formats |
| `guardrail reality` | Browser testing | ✅ Auto-installation |
| `guardrail ship` | Ship Check | ✅ Enhanced reporting |
| `guardrail fix` | AI-powered fixes | ✅ Guided suggestions |
| `guardrail init` | Project initialization | ✅ Template system |

### **Authentication:**
```bash
guardrail auth --key YOUR_API_KEY
guardrail auth --status
guardrail auth --logout
```

### **Advanced Scanning:**
```bash
# Enhanced vulnerability scanning
guardrail scan:vulnerabilities-enhanced --path ./project

# OSV integration
guardrail scan:vulnerabilities-osv --path ./project

# Compliance frameworks
guardrail scan:compliance --framework SOC2 --path ./project
```

### **Reality Mode:**
```bash
# Basic browser testing
guardrail reality --url https://example.com

# User journey flow
guardrail reality --url https://example.com --flow user-journey

# Custom flow with evidence collection
guardrail reality --url https://example.com --flow checkout --evidence
```

### **Ship Check:**
```bash
# Standard ship check
guardrail ship --path ./project

# Pro ship check with comprehensive scanning
guardrail ship:pro --path ./project --include-compliance
```

### **Project Management:**
```bash
# Initialize with template
guardrail init --path ./project --template startup

# Generate CI/CD pipeline
guardrail init --path ./project --ci github --format sarif
```
