# Non-Coder Features Guide

## 🎯 Overview

guardrail AI now includes features specifically designed to help non-coders communicate with their AI agent and understand their project better.

## ✨ Key Features

### 1. **Documentation Auto-Update Settings**

**Visual UI for configuring documentation updates:**
- Choose update frequency (Never, Daily, Weekly, On Change, Manual)
- Select which files to update (README, Quick Start, Scripts, API Docs)
- See last update status
- Update now button

**Benefits:**
- Never worry about outdated documentation
- AI agent won't create duplicate files
- Documentation always matches your code

### 2. **Duplicate File Detector**

**Automatically finds:**
- Exact duplicate files
- Similar files (80%+ similarity)
- Unused files

**Benefits:**
- Prevents AI agent from creating unnecessary files
- Keeps project clean
- Saves space

### 3. **Non-Coder Helper**

**Visual question suggestions:**
- Project Health - "Check if my project is healthy"
- Documentation - "Update my README"
- Files & Organization - "Find duplicate files"
- API & Endpoints - "Show all my API endpoints"
- Components - "Show unused components"
- Code Quality - "Polish my code"

**Benefits:**
- Don't know what to ask? Pick a category
- Pre-written questions you can use
- Learn what's possible

### 4. **Project Health Visualizer**

**Visual representation:**
- Overall health score (0-100)
- Category breakdown (Code Quality, Documentation, Tests, Security, Performance)
- Issues list with icons
- Recommendations

**Benefits:**
- See project status at a glance
- Understand what needs fixing
- Color-coded for easy understanding

## 🚀 Usage

### Configure Documentation Updates

1. Open dashboard
2. Go to "Documentation" tab
3. Choose update frequency
4. Select files to update
5. Click "Update Now" to test

### Check for Duplicates

```bash
guardrail check-duplicates
```

### Use Helper

1. Open dashboard
2. Click "AI Helper"
3. Pick a category
4. Click a suggested question
5. Question is sent to AI agent

### View Project Health

1. Open dashboard
2. Go to "Health" tab
3. See overall score and breakdown
4. Review issues and recommendations

## 🔧 Configuration

### Documentation Update Frequency

- **Never** - Manual updates only
- **Daily** - Update every day
- **Weekly** - Update once a week (Recommended)
- **On Change** - Update when code changes
- **Manual** - Update when you click

### Files to Update

- ✅ README.md - Main project documentation
- ✅ QUICK-START.md - Quick start guide
- ✅ SCRIPTS.md - Available scripts
- ✅ API-DOCS.md - API endpoint documentation
