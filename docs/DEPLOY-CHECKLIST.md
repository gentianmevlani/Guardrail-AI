# guardrail Deploy Checklist

**Version:** 1.0.0  
**Last Updated:** 2026-01-01

## Pre-Deploy Checklist

### 1. Build Verification
- [ ] `pnpm build` completes without errors
- [ ] `pnpm lint` passes (0 errors)
- [ ] `pnpm test` passes (all tests green)
- [ ] TypeScript compiles without errors

### 2. Environment Configuration
- [ ] `.env` file exists with production values
- [ ] `DATABASE_URL` points to production database
- [ ] `JWT_SECRET` is a strong 32+ character random string
- [ ] `NODE_ENV=production`
- [ ] All required secrets are set (see `env.example`)

### 3. Security Audit
- [ ] No hardcoded secrets in codebase (`grep -r "sk_live\|api_key="`)
- [ ] `.gitignore` includes all sensitive patterns
- [ ] HTTPS enforced on all endpoints
- [ ] CORS configured for production domains only
- [ ] Rate limiting enabled on API

### 4. CLI Verification
```bash
# Run these commands and verify outputs:
guardrail --help              # Shows help with ship commands
guardrail doctor              # All checks pass ✅
guardrail ship check --ci     # Exit 0 = SHIP
```

### 5. MCP Server Verification
```bash
# Start MCP server and verify connection
guardrail mcp
# Test from editor: run "get_status" tool
```

### 6. Dashboard Verification
- [ ] `/dashboard` loads without errors
- [ ] `/ship-check` displays gates properly
- [ ] `/runs` shows run history with filters
- [ ] `/runs/[id]` shows run details with all tabs
- [ ] `/policies` allows CRUD operations
- [ ] `/billing` displays pricing tiers
- [ ] Command palette opens with Ctrl+K

### 7. GitHub Integration
- [ ] `.github/workflows/guardrail-check.yml` is committed
- [ ] GitHub App is installed on target repos
- [ ] PR checks run and post comments
- [ ] SARIF uploads to Security tab

### 8. Landing Demo
- [ ] Run `.\scripts\update-landing-demo.ps1`
- [ ] Verify `apps/web-ui/public/demos/` contains:
  - [ ] `reality.webm` (video)
  - [ ] `reality-mode-result.json` (report)
  - [ ] `replay.json` (sanitized replay)

## Rollback Procedure

### 1. Identify Issue
- Check error logs: `railway logs` or Netlify dashboard
- Identify failing component (API, Web UI, CLI)

### 2. Rollback Steps
```bash
# Railway
railway rollback

# Netlify
netlify rollback

# Docker
docker pull guardrail:previous
docker tag guardrail:previous guardrail:latest
docker push guardrail:latest
```

### 3. Notify Team
- Post in #guardrail-ops Slack channel
- Update status page if public-facing

## Exit Codes Reference

| Code | Meaning | Action |
|------|---------|--------|
| 0 | SHIP - All checks passed | Safe to deploy |
| 1 | NO-SHIP - Violations found | Fix blockers |
| 2 | Misconfiguration | Check .guardrailrc |
| 3 | Runtime error | Check logs |
