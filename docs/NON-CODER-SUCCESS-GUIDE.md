## 🎯 Complete Workflow

### Step 1: Start Your Project

```bash
npx ai-agent-guardrails wizard
```

**Answer:**
1. Project name: `my-app`
2. What are you building? → Select "Full-Stack Web Application"
3. Framework? → Press Enter (uses Next.js)
4. Database? → Press Enter (uses PostgreSQL)
5. Done!

**Result:** Your entire project is built and ready!

### Step 2: Lock Your Design

```bash
npm run design-system
```

**Answer:**
1. Choose theme → Select "Modern" (or Dark, or Elegant)
2. Customize colors? → No (or Yes if you want)
3. Done!

**Result:** Your design is locked. AI can't create mismatched components!

### Step 3: Build Your App

Now when you ask AI to create components:

```
You: "Create a card component"
AI: Uses your locked design system
→ Card matches everything automatically!

You: "Create another card"
AI: Uses your locked design system
→ Matches the first card perfectly!

You: "Create a button"
AI: Uses your locked design system
→ Matches your cards and everything else!
```

**Result:** Everything is consistent! No more mismatched designs!

## 🎨 How Design Locking Works

### Before (Without Lock)
```
You: "Create a card"
AI: Creates card with random color #A1B2C3
→ Doesn't match anything

You: "Create another card"
AI: Creates card with different color #F4F5F6
→ Still doesn't match

You: "Create a button"
AI: Creates button with yet another color #123456
→ Nothing matches!
```

### After (With Lock)
```
You: "Create a card"
AI: Must use colors.primary from your locked system
→ Card uses #3B82F6 (your primary color)

You: "Create another card"
AI: Must use colors.primary from your locked system
→ Card uses #3B82F6 (same color - matches!)

You: "Create a button"
AI: Must use colors.primary from your locked system
→ Button uses #3B82F6 (matches everything!)
```

**Result:** Everything matches perfectly! ✅
